# 1.
Отличие C от C++  в том, что в Си структура могла хранить только характеристики, а поведения для них задавалось через внешние функции.

В C++ в структурах можно хранить как и характеристики, так и их поведение(функции)

**Инкапсуляция** - это объединение функций(поведения) и характеристик(полей) с возможностью регламентировать доступ к членам класса

**Класс** задает новый тип данных. 

Переменная данного типа данных называется **объектом**.

Также объекты класса называются **экземплярами класса**.

Характеристики класса - его **поля**.

**Методы класса** - это функции, принадлежащие классу, которые описывают поведение его объектов, а также могут изменять поля класса

Есть три спецификатора доступа:
**public** - открыт для всех доступ
**private** - открыт только для членов класса
**protected** - открыт как членам класса, так и наследникам

Различие между **struct** и **class** в доступе по умолчанию заключается в том, что в class по умолчанию стоит **private**, в **struct** по умолчанию стоит **public**;

Рекомендации по проектированию класса: 
Поля и служебные функции - **private**
Интерфейсные функции - **public**

Ключевым словом **this** обозначается указатель на объект класса в функциях-членах класса

Указатель на объект передается неявно во все функции, которые может вызвать только экземпляр класса

Если коллизии имен не возникает, то указатель на объект можно опустить.

# 2.

**Конструктор** - это специальная функция, которая инициализирует объект данного класса

**Конструктор** предназначен для инициализации полей класса начальными значениями.

**Имя** конструктора должно совпадать с именем класса


конструктор со списком инициализаторов:
Point2D point(): x(10), y(10){};

При использовании обычного конструктора с параметрами, сначала все равно вызывается конструктор по умолчанию, а лишь потом в теле конструктора значения полей класса будут перезаписаны на значения переданных аргументов конструктора, т.е произойдет двойная работа.

В случае использования конструктора со списком инициализаторов, поля инициализируются напрямую в момент создания.

Все функции, находящиеся в заголовочном файле будут автоматически помечаться модификатором **inline**

Конструктор без параметров называется конструктором **по умолчанию**.
Если в классе явно не описан ни один конструктор, то компилятор сгенерирует автоматически(неявно) конструктор по умолчанию.  

Если мы хотим в дальнейшем создать массив объектов данного класса, то необходимо будет **явно описать конструктор по умолчанию**!


**Конструктор копирования предназначен** для того чтобы определить, как создать новый объект как копию существующего.

**Конструктор копирования предназначен для создания независимой** копии ресурса(глубокое копирование), а не простого копирования указателя(поверхностное копирование).

Если конструктор копирования не описан **явно**, то компилятор **сгенерирует его автоматически**. Такой конструктор копирования является конструктором с **побитовым копированием** переменных членов-класса, следовательно осуществляет **поверхностное копирование**.

Для классов, использующих динамическую память, необходимо явно создавать конструктор копирования


Случаи вызова конструктора копирования:
1. явное создание объекта-копии
2. вызов функции с передачей параметра по значению
3. возврат объекта из функции по значению
